↔ Arrastra la esquina inferior derecha para redimensionar

¿Cómo funciona?

Media queries responden al tamaño del viewport (la ventana del navegador). Se usan para hacer diseños responsivos:

@media (max-width: 600px) {
  .sections-list {
    padding: 1rem;
  }
}

Container queries responden al tamaño del contenedor padre, no del viewport. Esto permite componentes verdaderamente reutilizables:

.queries-container {
  container-type: inline-size;
}

@container (max-width: 300px) {
  .q-item {
    flex-direction: column;
  }
}

Esta página usa media queries para adaptar el layout en pantallas pequeñas, y container queries en el demo de arriba.

¿Cómo funciona?

Flexbox organiza elementos en una dirección principal (fila o columna). El contenedor flex controla cómo se distribuyen y alinean sus hijos:

.container {
  display: flex;
  justify-content: space-between; /* eje principal */
  align-items: center;           /* eje cruzado */
  gap: 1rem;
}

Los items pueden crecer o encogerse con flex-grow, flex-shrink y flex-basis:

.item {
  flex: 1; /* shorthand: grow=1 shrink=1 basis=0 */
}

El header de esta página y los encabezados de cada sección usan flexbox para alinear su contenido.

CSS Grid

Demo: Diferentes layouts de Grid

Header
Sidebar
Main
Aside
Footer

¿Cómo funciona?

CSS Grid permite crear layouts bidimensionales (filas y columnas simultáneamente):

.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-template-rows: auto 1fr auto;
  gap: 1rem;
}

Con grid-template-areas puedes nombrar regiones:

.grid {
  grid-template-areas:
    "header header header"
    "sidebar main aside"
    "footer footer footer";
}

La unidad fr distribuye el espacio disponible proporcionalmente.

¿Cómo funciona?

Las variables CSS (custom properties) se declaran con --nombre y se usan con var():

:root {
  --color-primary: hsl(220, 70%, 55%);
  --spacing: 1rem;
  --radius: 12px;
}

.card {
  background: var(--color-primary);
  padding: var(--spacing);
  border-radius: var(--radius);
}

calc() permite hacer aritmética con unidades mixtas:

.main {
  width: calc(100% - var(--sidebar-width));
  padding: calc(var(--spacing) * 2);
}

Toda esta página usa variables CSS para colores, espaciado y border-radius definidos en :root.

Elemento padre Hijo anidado

Pasa el cursor sobre el elemento para ver los estilos anidados en acción

¿Cómo funciona?

La anidación nativa de CSS permite escribir selectores hijos dentro del bloque del padre, similar a SASS/LESS pero sin preprocesador:

.parent {
  color: blue;

  .child {       /* .parent .child */
    color: red;
  }

  &:hover {      /* .parent:hover */
    color: green;
  }

  &.active {     /* .parent.active */
    font-weight: bold;
  }
}

El & representa al selector padre. Es obligatorio para pseudo-clases y para combinar con el selector padre.

Todo el archivo styles.css de esta página usa anidación nativa para organizar las reglas.

Transformaciones

Demo: Hover para 3D, botones para transformar

Pasa el cursor aquí
Caja transformable

¿Cómo funciona?

Las transformaciones CSS modifican la forma, posición y orientación visual de un elemento sin afectar el layout:

.element {
  transform: rotate(45deg) scale(1.2);
  transform-origin: center center;
}

/* 3D con perspectiva */
.container {
  perspective: 800px;
}
.element {
  transform: rotateY(30deg)
             rotateX(10deg);
}

Funciones principales: translate(), rotate(), scale(), skew(). Se pueden combinar en una sola declaración.

El card 3D de arriba usa rotateX y rotateY al hacer hover.

Transiciones

Demo: Carrera de timing functions

ease-out

¿Cómo funciona?

Las transiciones animan el cambio entre dos estados de una propiedad CSS de forma suave:

.element {
  background: blue;
  transition: background 0.3s ease,
              transform 0.3s ease;
}

.element:hover {
  background: red;
  transform: scale(1.1);
}

Propiedades clave:

  • transition-property — qué propiedad animar
  • transition-duration — cuánto dura
  • transition-timing-function — la curva de velocidad
  • transition-delay — cuánto esperar antes de empezar

Animaciones

Demo: Animaciones con @keyframes

Bounce
Pulse

CSS Moderno

¿Cómo funciona?

Las animaciones CSS permiten cambios complejos y multi-paso usando @keyframes:

@keyframes bounce {
  0%, 100% {
    transform: translateY(0);
  }
  50% {
    transform: translateY(-30px);
  }
}

.ball {
  animation: bounce 0.6s
             ease-in-out infinite;
}

A diferencia de las transiciones, las animaciones:

  • Pueden tener múltiples pasos (keyframes con %)
  • Se ejecutan automáticamente (sin hover)
  • Pueden repetirse infinitamente
  • Tienen control de dirección (normal, reverse, alternate)
